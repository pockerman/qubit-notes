## TCP 101

TCP is a communication ptocol typically found at the transport layer of a layered communication architecture. It operates on top of the IP
and guarantees that mesages arrive in order and without gaps duplication or corruption [1]. TCP also implemets a number of congenstion control
algorithms to avoid overwhelming the network or the receiver.

The most important trait of TCP is its reliabiltiy. The way TCP achieves its reliability is by partiotioning a byte stream into discrete packets also
called segments [1]. Every segment that is sent has to acknowledged by the receiver. Segments are sequentially numbered which allows for the receiver to detect
holes and duplicates. TCP reliability is also based on retransmittion that  occurs when the receiver does not return an acknowledgement for a sent segment
within a specified time window [1]. The receiver can use a checksum to verify the integrity of the delivered data.

A TCP connection has many states but the most important ones are:

- Opening state
- Established state
- Closing state

During the opening state a connection is being cretaed. A server must be listening  for connection nrequests from clients before a connection is seatblished.
TCP uses a three-way handshake in order to create a new connection [1]:

1. The sender picks a random sequence number say $x$ and sends a SYN segment to the receiver.
2. The receiver increments the sequence $x$ and chooses a random sequence number $y$ and sends back a SYN/ACK segment.
3. The sender increments both sequence numbers and replies with an ACK segment and the first bytes of the applicaiton data.

The handshake mechanism involves a full round-trip where no application data is sent. Its bandwidth is essentially zero until a conneciton has been opened.
Thus the lower the round-trip is the faster the conneciton can be established. This is the reason why you want to put servers closer to the clients and reusing 
connections already opened. Once the transmission is complete, the connection has to be closed and all the resources released by both sides otherwise resource leakage occurs.
Terminating a TCP connection involves a number of round-trips [1].

### Congestion and flow control

TCP implements a backoff mechanism to prevent the sender from overwhelming the receiver. The receiver uses some sort of buffer in order to
store the incoming TCP segments to be processed. The size of this buffer is communicated to the sender whenever the receiver communicates an ACK to the sender.
This approach is not too different to <a href="./2026-02-09-Upstream-Resiliency.md">rate-limiting</a> [1].

Additionally, TCP also guards against overwhelming the underlying network. This is done by estimating the avaialble bandwidth of the network and maintaining
a _congestion window_ which represents the total number of outsatnding segments that can be sent without any ACK from the receiver [1].
Obviously, the small the congestin window, the fewer bytes are in-flight and therefore therefore the less bandwidth is utilised. When a new connection is established the congestion window size is et to some system default. It then increases exponentially for every segment that is ACK until an upper limit is reached [1]. Hence, we cannot utilise the available bandwidth
as soon as a connection is established. In fact under TCP the bandwidth utilised is given by [1]

$$
\text{Bandwidth} = \frac{\text{Window size}}{RTT}
$$ 

If a segment is lost, identified by a missed acknowledgement via a timeout, congestion avoidance kicks in which reduces the size of the congestion window. Thus, timeouts will decrease the window size whilst normal operation increases it up to predefined upper limit.

All in all TCP is a reliable and stable communication protocol that can suffer from high latency and lower bandwidth than what the underlying network is capable of delivering.
However, if reliablity and stability are not needed or do not make sense for the application at hand, protocols such as the <a href="https://www.sciencedirect.com/topics/computer-science/user-datagram-protocol">User Datagram Protocol or UDP</a> can be used. 

## Summary

This part discussed TCP; a transport-layer protocol in a layered network architecture, enables reliable communication between processes. TCP ensures ordered, complete, and error-free data delivery by splitting data into numbered segments, requiring acknowledgments, retransmitting lost packets, and verifying integrity with checksums. Connections are established using a three-way handshake (SYN, SYN/ACK, ACK), which introduces latency due to a full round-trip before data transfer begins, and are later closed through additional exchanges. TCP also manages flow control by adjusting to the receiver’s buffer capacity and prevents network congestion using a dynamically sized congestion window that grows with successful transmissions and shrinks when packet loss is detected. While TCP provides strong reliability and stability, these mechanisms can limit performance in terms of latency and bandwidth utilization, which is why less strict alternatives like UDP may be preferred in scenarios where reliability is less critical.
